# Front-end

## 이런 일을 맡고 있어요

SmartEditor ONE 플랫폼\(이하 SE ONE 플랫폼\)은 NAVER 전사의 콘텐츠 저작 플랫폼으로서 다양한 서비스에 콘텐츠 생산/소비 환경을 제공하고 있으며, 월 단위 약 6,000,000건의 콘텐츠가 SE ONE 플랫폼을 통해 만들어지고 있습니다. Front-end 개발자는 SE ONE 플랫폼의 JavaScript SDK와 Web UI 프레임워크를 개발합니다. 보다 나은 사용 경험과 안정적인 성능을 위해 UX/UI 설계와 Back-end 개발자, iOS 개발자, Android 개발자, 테스트 엔지니어와 함께 고민하고 제품을 만들어 갑니다.

네이버의 다른 Front-end 개발 조직과 달리 SE ONE 플랫폼의 Front-end 개발 조직은 빠르고 정교한 WYSIWYG 편집기를 만들기 위해 JavaScript 개발자와 Web UI 개발자가 함께 있으며 웹 문서 편집과 관련한 다양한 특허를 보유하고 있습니다. 이러한 기술력을 바탕으로 SmartEditor1부터 SmartEditor ONE까지 많은 버전의 스마트에디터를 만들었습니다.

* SmartEditor1, SmartEditor2: JavaScript 라이브러리
* SmartEditor3: 데스크톱 웹과 모바일 웹에 최적화된 SPA 형태의 스마트에디터와 문서 뷰어 라이브러리
* SmartEditor ONE: 이전 버전의 모든 장점을 통합하고 쉬운 적용과 커스터마이징이 가능한 스마트에디터와 문서 뷰어 SDK

## 우리 같이 해봐요

* SE ONE 플랫폼 JavaScript SDK 개발
* SE ONE 플랫폼 Web UI 프레임워크 개발
* 개발자 가이드, API 레퍼런스 기술 문서 작성

## 우리는 이렇게 일해요

* 최적화된 사용자 경험의 멀티플랫폼 WYSIWYG 편집기를 만들기 위해 UX/UI 설계와 Back-end 개발자, iOS 개발자, Android 개발자와 긴밀하게 협업합니다.
* JavaScript 생태계의 언어와 프레임워크, 라이브러리 등의 깊은 이해를 바탕으로 제품을 만들어 갑니다.
  * ES6
  * Webpack, Babel
  * Node.js, MongoDB
* 웹 애플리케이션 프레임워크를 활용하여 생산성을 높이고자 노력합니다.
  * React
  * Mobx
  * ESlint, Karma, Mocha, Chai, Sinon, Enzyme
* 다양한 환경에서 최적화된 사용자 경험을 제공하기 위해 성능 최적화를 꼭 챙깁니다.
  * 클라이언트 성능 최적화를 위한 브라우저 로딩/렌더링 최적화 작업
  * Node.js 기반 서버 성능 최적화 작업
  * 반응형 웹
* 네이버와 글로벌 서비스의 장애 예방 및 가용성, 확장성 확보를 위해 인프라 엔지니어링을 고민하고 해결합니다.
  * Sentry, CI/CD
  * Kubernetes, Docker

### 개발문화

#### Agile, DevOps

2주 단위의 스프린트와 데일리 스크럼을 가집니다. 하나의 스프린드 단위로 정기 릴리스 프로세스를 진행하며, 합의로 도출된 코드 컨벤션과 그라운드 룰을 따릅니다. 매 PR의 코드 리뷰는 반드시 필요하며, 페어 코딩을 권장합니다. 품질 유지를 위해 테스트 코드를 작성하며, 모든 개발 과정은 자동화된 CI/CD와 연계되어 있습니다.

#### 협업 + 커뮤니케이션

FE 개발자는 UI 접점을 담당하기 때문에 많은 유관조직 \(기획/UX/BE/QA 등\)들과의 원활한 협업능력이 숙명입니다. 거기에 더해 스마트에디터 특수성으로 인해 에디터를 적용하는 서비스개발조직뿐만 아니라 포토인프라/동영상인프라 등 다양한 기능 연동을 위해 타 조직과의 긴밀한 협업이 필요합니다.

#### 자주성

부족한 기능이나 비효율적인 프로세스를 찾아서 개선할 수 있는 분들과 함께 성장하고 싶습니다.

스마트에디터는 조직 특성상 서비스측에 안정적인 제품을 제공해야 하는 입장입니다.  
최신트렌드에 민감한 FE 개발 영역에서 새로운 시도를 해보기 어려운 환경일 수도 있습니다.  
하지만 스마트에디터 개발팀은 팀원들의 목소리를 적극 수용하고 있습니다.

* 사진검색기능, 이미지리사이즈기능 등 일부 기능들은 개발팀의 역제안으로 추가된 케이스이며 최근에는 개발자 주도로 진행된 번역기능도 탑재될 예정입니다.
* 제품의 직접적인 기능이 아니더라도 FE 신기술이나 Nodejs 서버개발경험에 대한 니즈를 충족하기 위해 사내 데모서비스를 구축하고 운영하고 있습니다.
* 오픈소스개발경험을 해보고 싶으시면 이미 오프소스화 되어 있는 스마트에디터2 를 발판 삼아볼 수도 있습니다.

#### 사용자 친화적 기술력

기술을 대하는 관점도 사용자친화적으로 바라보길 기대합니다.

> SmartEditor 는 엔드유저뿐만 아니라 에디터를 적용하는 서비스개발자들도 사용자인 셈입니다.

* 엔드유저에게 편리한 UI 를 제공하는 것은 기본이고
* 서비스개발자관점에서 얼마나 안정적이고 편리한 API 를 제공할 수 있을지를 고려한 기술적인 설계와 접근이 필요합니다.

## 이런 기회가 있습니다

* 네이버와 글로벌 서비스의 사용자 접점이 되는 UX/UI 설계부터 BFF\(Back-end for Front-end\)까지 경험할 수 있는 기회가 있습니다.
* 네이버와 글로벌 서비스의 사용자를 대상으로 하는 기능을 설계하고 개발할 수 있는 기회가 있습니다.
* Front-end 기술의 끝판왕인 WYSIWYG 웹 애플리케이션을 설계하고 개발할 수 있는 기회가 있습니다.

## 이미 당신은 준비되어 있네요

* JavaScript 애플리케이션 개발 경험
* HTML, CSS 등 W3C 웹 표준 규격에 대한 이해와 개발 경험
* React/Vue/Angular 기반 SPA 개발 및 운영 경험
* 객체지향/함수형 프로그래밍의 기본 지식 보유
* 웹 접근성, 렌더링 성능 최적화 및 다양한 환경의 크로스 브라우저 개발 경험
* UX/UI 설계, 개발자와 원활한 커뮤니케이션이 가능하신 분
* 코드 리뷰와 테스트 코드 작성 등의 좋은 코드를 위한 활동에 적극적인 분
* 신기술에 대한 호기심을 가지고, 이를 이용하는 것을 즐기는 분
* 혼자 빨리 가는 것 보다 늦더라도 동료와 함께 멀리 갈 수 있는 분
* 회고 등의 적극적인 개선 활동을 통해 어제보다 더 나은 오늘을 꿈꾸는 분

* WYSIWYG 편집기 설계/구현/운영한 경험이 있는 분
* 크로스 브라우저 환경 경험이 있는 분
* JavaScript, 브라우저 렌더링 성능 개선 경험이 있는 분
* 마크업 개발 환경 구성, 스프라이트 자동화 등 빌드 환경 구성 경험이 있는 분
* CSS Pre Processor 고급 기능을 활용하여 논리적인 개발이 가능한 분
* 테스트 코드 작성 및 자동화에 풍부한 경험이 있는 분
* 효율적인 UI 설계에 대한 풍부한 지식과 경험을 가진 분
